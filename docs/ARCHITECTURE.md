# JamTheory Architecture

## 1. Overview

JamTheory is a three-part application:

1. **Frontend** — React + Vite SPA hosted on Firebase Hosting. Reads and writes
   Firestore directly (subject to security rules) and calls the backend via
   API Gateway for Gemini-backed endpoints.
2. **Backend** — two Python Cloud Functions (2nd gen) fronted by API Gateway:
   - `generate-masterclass` — turns a YouTube video ID + skill level into the
     full masterclass payload (key, theory, chords, scales, target notes,
     3 AlphaTex licks).
   - `regenerate-licks` — re-rolls only the 3 vocabulary licks for a cached
     track without overwriting the rest of the document.
3. **Infrastructure** — Terraform-managed GCP resources (Firestore, Cloud
   Functions, API Gateway, Secret Manager, Firebase Hosting, IAM).

See `jam_feature.md` for product intent. This document focuses on *how*.

## 2. Component details

### 2.1 Frontend (React + Vite)

- **Role:** single-screen desktop dashboard. Left column is the Practice Zone
  (URL input, YouTube player, stopwatch, "Finish & Log" button). Right column
  is the Masterclass Content (theory, rhythm, lead, vocabulary).
- **Hosting:** Firebase Hosting.
- **State:** local `useState`/`useReducer`. No global state library.
- **Firebase SDK:** modular v10+, isolated behind `src/firebase/*` and
  `src/services/*`. Components never import from `firebase/*` directly.
- **Auth:** Firebase Auth, Google Sign-In. On first login the user is created
  with `status: "pending"`. An admin flips it to `"active"` via the Firebase
  Console (identical to the Daskalo reference). A skill-level onboarding
  modal sets `users/{uid}.skillLevel` on first run.
- **Backend calls:** raw `fetch()` with the Firebase Callable wire protocol:

      Request body:  { "data": { ...args..., "idToken": "<firebase-jwt>" } }
      Success body:  { "result": { ... } }
      Error body:    { "error": { "status": "...", "message": "..." } }

  The ID token is also sent as `Authorization: Bearer <token>` for local dev.
  URLs come from `src/env.ts` and `src/environments/environment.prod.ts`
  (the prod file is generated by `deploy.sh --hosting`).
- **AlphaTab:** `@coderline/alphatab` is lazy-loaded via dynamic `import()`
  inside `components/Masterclass/AlphaTabRenderer.tsx` so it does not bloat
  the initial bundle.

### 2.2 Backend (Cloud Functions 2nd gen + API Gateway)

- **Deployment:** one function per entry-point Python file.
- **Auth (layered):**
  1. API Gateway validates the Firebase JWT at the edge
     (`issuer: https://securetoken.google.com/{project_id}`,
      `audience: {project_id}`). Missing/invalid tokens → `401`.
  2. Cloud Functions are `--no-allow-unauthenticated`. Only the
     `api-gateway-sa` service account can invoke them via `roles/run.invoker`.
  3. Function code re-verifies the Firebase ID token (defence-in-depth).
- **Wire protocol:** Firebase Callable (`{"data": {...}}` / `{"result": {...}}` /
  `{"error": {...}}`).
- **Functions:**
  - `generate_masterclass_fn` (`fn_generate_masterclass.py`):
    scrape YouTube metadata → regex-check for chords → if absent, return
    `{ "needsChords": true }` → call Gemini with the full system prompt and
    a Pydantic-derived `response_schema` → return the masterclass JSON.
    The function does NOT write to Firestore — the frontend caches the
    response in `backing_tracks/{videoId}/variants/{skillLevel}`.
  - `regenerate_licks_fn` (`fn_regenerate_licks.py`):
    smaller prompt; input is `{ key, genre, skillLevel }` and output is
    `{ licks: [Easy, Medium, Hard] }` (AlphaTex only). Does NOT write to
    Firestore; the UI swaps the in-memory licks.
- **Shared helpers (`callable_helpers.py`):** token verification, request
  parsing, CORS, Callable response formatting.
- **Service accounts:**
  - `api-gateway-sa` — held by API Gateway, has `roles/run.invoker`.
  - `cf-runtime-sa` — attached to both functions, has
    `roles/aiplatform.user`, `roles/datastore.user`,
    `roles/firebase.sdkAdminServiceAgent`, `roles/storage.objectViewer`,
    `roles/secretmanager.secretAccessor`.

### 2.3 Secrets

The YouTube Data API v3 key is stored in Secret Manager
(`youtube-api-key`) and mounted into both Cloud Functions via Terraform's
`secret_environment_variables` as `YOUTUBE_API_KEY`.

## 3. Data flow

### 3.1 Generate Masterclass (cache miss)

1. User pastes a YouTube URL. Frontend extracts the video ID.
2. Frontend reads `backing_tracks/{videoId}/variants/{skillLevel}`. If it
   exists → render immediately (step 7).
3. Frontend calls `{api_gateway_url}/generate-masterclass` with
   `{ youtubeVideoId, skillLevel, userChords? }`, sending the Firebase ID
   token both in `data.idToken` and in `Authorization: Bearer`.
4. API Gateway validates the JWT and proxies to the Cloud Function.
5. The function:
   - fetches the video's title + description via YouTube Data API v3;
   - runs a regex fallback — if no chords are detected AND `userChords` is
     absent, it returns `{ needsChords: true }` (frontend prompts the user,
     then retries);
   - calls Gemini (`gemini-3.1-pro-preview`) with the system prompt from
     `jam_feature.md` §6 and a strict `response_schema`;
   - returns the parsed JSON to the caller.
6. Frontend writes the result to
   `backing_tracks/{videoId}` (video-level metadata, create-only) and
   `backing_tracks/{videoId}/variants/{skillLevel}` (masterclass payload,
   create-only). Security rules enforce schema shape and immutability.
7. UI renders the two-column dashboard.

### 3.2 Regenerate Licks

1. User clicks "Generate New Licks" under the Vocabulary section.
2. Frontend calls `{api_gateway_url}/regenerate-licks` with
   `{ key, genre, skillLevel }`.
3. API Gateway validates the JWT, proxies to the Cloud Function.
4. The function calls Gemini with the compact lick-only prompt and returns
   3 AlphaTex licks.
5. Frontend swaps the in-memory licks. **It does not overwrite the cached
   `variants/{skillLevel}` document** (per spec §6).

### 3.3 Log Practice Session

1. User clicks "Finish & Log Session".
2. Frontend writes `users/{uid}/practice_logs/{auto}` with
   `{ youtubeVideoId, date, minutesPlayed }`. No backend call needed.

## 4. Local development

Run `./dev.sh` from the repo root. It starts three processes:

1. **Firebase Emulator Suite** — Auth (9099), Firestore (8081), UI (4001).
2. **FastAPI backend** (`uvicorn main:app --reload` on `:8000`). `main.py`
   bundles both Cloud Function handlers behind plain FastAPI endpoints via
   a `_FlaskRequestShim`. `main.py` is **not deployed**.
3. **Vite dev server** on `:5173`.

### Environment URLs

| Environment | `generateMasterclassUrl`                 | `regenerateLicksUrl`                  |
|-------------|------------------------------------------|---------------------------------------|
| Local       | `http://localhost:8000/generate-masterclass` | `http://localhost:8000/regenerate-licks` |
| Production  | `{api_gateway_url}/generate-masterclass` | `{api_gateway_url}/regenerate-licks`  |

## 5. Deployment

All cloud infrastructure is managed by Terraform in `infra/`.

### First-time setup

```bash
export PROJECT_ID=your-gcp-project-id
./bootstrap.sh                              # create TF state bucket, init
cp infra/terraform.tfvars.example infra/terraform.tfvars
# fill in values — project_id, youtube API key ref, etc.
./deploy.sh                                 # infra + hosting
```

### Subsequent deploys

```bash
./deploy.sh --infra      # infra only (rebuilds backend zip + terraform apply)
./deploy.sh --hosting    # frontend build + firebase deploy
./deploy.sh              # both
```

### What `deploy.sh` does

- **`--infra`**: zips `backend/` (excludes `.venv/`, `tests/`, `main.py`,
  `__pycache__`, `.env*`) → uploads to GCS → `terraform apply`. Cloud
  Functions get a new revision only when the zip hash changes.
- **`--hosting`**: reads Terraform outputs (API Gateway URL, Firebase SDK
  config) → generates `frontend/src/environments/environment.prod.ts` →
  `npm run build` → `firebase deploy --only hosting,firestore`.

### Terraform resources

| File                     | Resources                                                    |
|--------------------------|--------------------------------------------------------------|
| `infra/main.tf`          | Providers, GCS backend                                       |
| `infra/apis.tf`          | All required `google_project_service` API enablements        |
| `infra/firestore.tf`     | Firestore database (NATIVE mode)                             |
| `infra/iam.tf`           | `api-gateway-sa`, `cf-runtime-sa`, IAM bindings              |
| `infra/secrets.tf`       | `youtube-api-key` Secret Manager secret                      |
| `infra/functions.tf`     | CF source bucket, source zip object, 2x `cloudfunctions2_function` |
| `infra/api_gateway.tf`   | API, config, gateway (Firebase JWT, CORS)                    |
| `infra/openapi.yaml.tpl` | OpenAPI 2.0 spec rendered with actual function URLs          |
| `infra/firebase_hosting.tf` | Firebase Web App + Hosting site                           |
| `infra/outputs.tf`       | API Gateway URL, Firebase SDK config                         |
